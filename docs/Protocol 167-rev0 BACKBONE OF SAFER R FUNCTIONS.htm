3.1.  Color code explanation

 

YOUR_PACKAGE                     strings that must be modified in your function in development.

check                                       optional sections:

 

Section

Required action

check of lib_path

either add lib_path argument in your developed function, indicating a specific R library path. Otherwise, empty the section.

check of the required functions from the required packages

write all function names present in your code, from any non basic package (packages that need to be imported in the working environment). Functions belonging to the same package as your function in development must also be indicated, still to avoid the same function name in another package. Otherwise, empty the section.

arg with no default values

write all the arguments that have no default values. Otherwise, empty the section.

management of NULL arguments

write all the arguments that cannot be NULL. Otherwise, empty the section.

management of "" in arguments of mode character

write all the arguments with string values that cannot be "". Otherwise, empty the section.

reserved words

write reserved words for your function (for instance in a column name of a data frame added to an imported data frame). Otherwise, empty the section.

code that protects set.seed() in the global environment

keep the code if a seed is used in random instructions. Otherwise, empty the section.

 

check                                       sections that would need to be rewritten in the following cases:

In the "management of NA arguments" section, if at least one of the argument of your developed function can be NA only.

3.3.  Code inside the main code of your function

This section indicates how to add some sections in your main code, if you have kept the 1) "reserved words" and 2) "warning initiation" sections indicated in paragraph 3.2 above.

3.3.1.   Reserved words

If ever you have reserved words in your main code, then you have kept the "reserved words" section in the backbone before the main code (see section 3.2). This example of what can be written in the main code section is given for a list named "data" that must not contain any reserved word in any of its i1 compartments:

 

      #### main code

 

      ######## reserved word checking

      if(base::any(base::names(x = data[[i1]]) %in% reserved_words, na.rm = FALSE)){

              tempo_cat <- base::paste0("ERROR IN ", function_name, base::ifelse(test = base::is.null(x = package_name), yes = "", no = base::paste0(" OF THE ", package_name, " PACKAGE", collapse = NULL, recycle0 = FALSE)), "\nCOLUMN NAMES OF ", base::ifelse(test = base::length(x = data1) == 1, yes = "data1", no = base::paste0("data1 NUMBER ", i1, collapse = NULL, recycle0 = FALSE)), " ARGUMENT CANNOT BE ONE OF THESE WORDS\n", base::paste(reserved_words, sep = " ", collapse = " ", recycle0 = FALSE), "\nTHESE ARE RESERVED FOR THE ", function_name, " FUNCTION", collapse = NULL, recycle0 = FALSE)

              base::stop(base::paste0("\n\n================\n\n", tempo_cat, "\n\n================\n\n", collapse = NULL, recycle0 = FALSE), call. = FALSE, domain = NULL)

      }

      ######## end reserved word checking

 

      #### end main code

 

3.3.2.   Warning

If ever you need to return a warning, then you have kept the "warning initiation" section in the backbone before the main code (see section 3.2), and this is how you should write your warning, each time you need it, as well as how you have to add the "warning output" section at the end of the code :

 

      #### main code

 

      ######## warning

      warn_count <- warn_count + 1

      tempo_warn <- base::paste0("(", warn_count,") THE FIRST FOR & WHILE LOOP STEPS HAVE BEEN TOO FAR AND SUBSEQUENT LOOP STEPS WILL NOT RUN", collapse = NULL, recycle0 = FALSE)

      warn <- base::paste0(base::ifelse(test = base::is.null(x = warn), yes = tempo_warn, no = base::paste0(warn, "\n\n", tempo_warn, collapse = NULL, recycle0 = FALSE)), collapse = NULL, recycle0 = FALSE)

      ######## end warning

 

      #### end main code

 

      #### output

      base::return(output)

      #### end output

 

      #### warning output

      if( ! base::is.null(x = warn)){

              base::on.exit(expr = base::warning(base::paste0("FROM ", function_name, base::ifelse(test = base::is.null(x = package_name), yes = "", no = base::paste0(" OF THE ", package_name, " PACKAGE", collapse = NULL, recycle0 = FALSE)), "\n\n", warn, collapse = NULL, recycle0 = FALSE), call. = FALSE, immediate. = FALSE, noBreaks. = FALSE, domain = NULL), add = FALSE, after = TRUE)

             }

             base::on.exit(expr = base::options(warning.length = ini_warning_length), add = TRUE, after = TRUE)

      #### end warning output
